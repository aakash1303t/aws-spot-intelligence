# EC2 Spot Price Intelligence

Forecasting and cost-optimization on AWS EC2 spot prices.
This repo currently covers the **data foundation** and **rung 1 of the forecasting core**.

## Project layout

```
spot-intel/
├── requirements.txt
├── data/
│   └── spot_prices.parquet        # generated by step 1 (not committed)
├── outputs/                       # plots + result CSVs land here
└── src/
    ├── generate_synthetic.py      # 1. make realistic synthetic spot data
    ├── ingest_aws.py              # 1b. (optional) pull REAL spot data via boto3
    ├── features.py                # 2. lag / calendar / rolling features (no leakage)
    ├── metrics.py                 # 3. MAE + MASE (seasonal-scaled)
    ├── models.py                  # 4. model ladder (naive → snaive → SARIMA → GBM)
    └── backtest.py                # 5. walk-forward harness + runner
```

## Setup (once)

```bash
python -m venv .venv && source .venv/bin/activate    # optional but recommended
pip install -r requirements.txt
```

## Step by step

All commands are run from the project root (`spot-intel/`).

### Step 1 — build the dataset
Creates `data/spot_prices.parquet` (~237k rows, 110 series, 90 days).
```bash
python src/generate_synthetic.py
```
*Optional — use real AWS data instead:* configure a free AWS account
(`aws configure`) and run `python src/ingest_aws.py`. It writes the **same
schema**, so every later step works unchanged.

### Step 2 — run the backtest (rung 1: naive baseline)
Holds out the last 14 days of every series, scores the naive forecast,
and writes `outputs/rung1_naive.csv`.
```bash
python src/backtest.py
```
Expected: mean MASE ≈ 0.33 — spot prices are highly persistent, so "last
value" is already a strong forecast. That 0.33 is the bar the next rungs must beat.

## How the pieces fit

`backtest.py` imports `load()` for the data, `metrics.py` for MASE, and the
forecasters in `models.py`. `features.py` feeds the gradient-boosting model
(used from rung 4 onward). Add a rung by writing its forecaster in `models.py`
and an `evaluate_*` function in `backtest.py` — the held-out split never changes.
